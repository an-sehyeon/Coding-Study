상수포인터와 포인터상수 정의와 차이?

 - 포인터 상수는 배열 생각 
 - 0번째 인덱스는 주소값을 가지고 있고 뒤의 값의 주소가 0번째 주소를 참조해 주소값이 할당되기때문에 주소값변경은 불가능 하지만 값변경은 가능
 - 변수명 앞에 const 붙으면 메모리값이 고정이 되기때문에 주소값 변경 불가능

 -상수포인터는 반대로 주소값을 변경 가능하지만 값변경은 불가능



char* const test = "abcde"; // 초기값, 주소값, 메모리값을 상수화, 즉 고정시킨다.

char* const test = "12345"; // X, Why? : 문자열 "12345"는 그 자체로 주소값을 가지고 있기 때문에

const char* test = "12345"	// 초기값, test가 가리키고 있는 값을 상수화, 즉 고정시킨다.

const char* test = "dlehdgk"; // O, Why? : 값이 변경된 것처럼 보이지만 사실 "dlehdgk"라는 문자열 자체가 새롭게 형성(메모리에 새로운 주소값 형성)돼서 그걸 test가 가리키는 형태
