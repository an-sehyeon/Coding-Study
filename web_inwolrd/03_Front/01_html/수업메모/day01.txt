1. web
(1) web의 기본 목적
    다른 컴퓨터에서 문서를 공유하거나 보려는 목적
(2) web의 구조
    인터넷을 활용하여 거미줄처럼 연결, WWW(World Wide Web)    
    웹 문서를 인터넷 상의 컴퓨터끼리 주고받는 네트워크 시스템    
(3) web의 구성
    웹 서버 - 웹 클라이언트

    1) 클라이언트
    사용자 인터페이스 담당
    웹 서버에 웹 문서를 요청

    2) 웹 서버
    웹 사이트를 탑재한 컴퓨터
    웹 문서, 이미지, 동영상 등 데이터를 저장

    3) 웹 컨테이너
    동적인 데이터 연산이 들어왔을 때 서버가 연산을 요청하는 곳
    연산이 끝나면 정제된 데이터를 서버에 돌려준다.

    4) WAS(Web Application Server)
    서버 + 컨테이너
    가장 유명한 것은 Apache Tomcat

    실제 시스템 구조로 가장 빈번히 사용되는 것은
    Web 서버 + WAS 서버 + DB 서버
    고정된 페이지(정적인 html, 이미지, 동영상)는 Web서버가,
    동적인 페이지(사용자마다 다른, 사용자의 입력에 대한 연산)는 WAS가
    처리하는 구조

2. 인터넷(Internet)
    웹의 개념이 나오기 전 만들어진 컴퓨터 연결 네트워크
    1969년 미 국방성 고등 연구 계획국(ARPA)
    여러 대학교와 계약 업체 사이의 컴퓨터 연결
    컴퓨터마다 고유의 주소(IP Address)를 부여하여 컴퓨터를 연결

3. WWW
    인터넷을 활용하는 응용 서비스 중 하나
    웹 서버와 웹 브라우저(클라이언트)로 
    구성되는 정보 전달 및 공유 서비스
    인터넷은 고속도로, 웹은 물류배송

4. Web Browser
    Netscape Navigator
    - 일반인도 사용 가능한 쉬운 GUI를 갖는 최초의 브라우저

    Internet Explorer
    - 1995년 Microsoft에서 개발
    - Windows 운영체계에 끼워서 배포, 순식간에 Netscape를 잠식

    Opera
    - 프로그램의 크기가 작고, 렌더링 빠름

    Safari
    - 2003년 Apple에서 개발
    - Mac os, 모바일 ios에서 실행

    FireFox
    - 2002년 개발, 웹 표준기구(W3C)의 표준에 가장 충실
    - Netscape가 개발하고 소스 공개

    Chrome
    - 2008년 구글에서 개발, 새로운 강자, 가장 많이 사용
    - 구글의 생태계, 개인화 등에 의해 유명해짐

    Microsoft Edge
    - Microsoft가 IE를 포기하고
      Chromium을 인수하여 확장 개발한 브라우저

    Brave Browser

5. 웹의 성공
    - Tim Berner-Lee의 아이디어
    - 1989년 웹 개념 제안 > 1990년 WWW 프로젝트 시작
    - 서버와 클라이언트로 동작하는 http 모델 창시
    - 웹 페이지는 텍스트 문서 > 편집기는 아무 것이나 사용 가능
    - http
        웹 브라우저가 웹 페이지를 다 수신받고 나면 
        웹 서버는 접속을 끊음
        웹 서버와 웹 브라우저의 역할(작업) 분담
        웹 서버의 낮은 부하로 많은 동접자 지원
